Ship to Mobile
🍎 iOS
🤖 Android

Four battle-tested methods to ship Volume on iPhone — from zero-code PWA to full App Store distribution.

1
PWA — Install via Safari (Easiest)
▶

A Progressive Web App lets users install this file directly to their iPhone home screen via Safari — no Xcode, no App Store needed. It runs full-screen like a native app.

Step 1: Create a manifest.json file next to your HTML:

// manifest.json
{
  "name": "Volume",
  "short_name": "Volume",
  "start_url": "./index.html",
  "display": "standalone",
  "background_color": "#0a0a0f",
  "theme_color": "#00e5ff",
  "orientation": "portrait",
  "icons": [
    { "src": "icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}

Step 2: Add these tags inside your HTML <head>:

<link rel="manifest" href="manifest.json">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style"
      content="black-translucent">
<meta name="apple-mobile-web-app-title" content="Volume">
<link rel="apple-touch-icon" href="icon-192.png">

Step 3: Create sw.js for offline support:

const CACHE = 'volume-v1';
const FILES = ['./index.html'];
self.addEventListener('install', e =>
  e.waitUntil(caches.open(CACHE).then(c => c.addAll(FILES)))
);
self.addEventListener('fetch', e =>
  e.respondWith(caches.match(e.request).then(r => r || fetch(e.request)))
);

Step 4: Register the service worker in your HTML, then host on HTTPS (GitHub Pages / Netlify). Open in Safari → Share → Add to Home Screen.

💡localhost won't work for PWA install — you need HTTPS. Use ngrok for quick local testing.
2
WKWebView — Xcode Native Wrapper
▶

Wrap the HTML file in a native iOS app using Xcode. Produces a real .ipa for TestFlight or App Store. Requires a Mac + Xcode 15+ + Apple Developer account ($99/yr).

import SwiftUI
import WebKit

struct WebView: UIViewRepresentable {
  func makeUIView(context: Context) -> WKWebView {
    let wv = WKWebView()
    if let url = Bundle.main.url(
      forResource: "index",
      withExtension: "html") {
      wv.loadFileURL(url,
        allowingReadAccessTo: url.deletingLastPathComponent())
    }
    return wv
  }
  func updateUIView(_ uiView: WKWebView, context: Context) {}
}

struct ContentView: View {
  var body: some View {
    WebView().ignoresSafeArea().preferredColorScheme(.dark)
  }
}
⚠️localStorage works fine inside WKWebView. Camera/mic require additional entitlements if added later.
3
Capacitor — Full Native Bridge
▶

Capacitor bridges native iOS APIs (haptics, push notifications, HealthKit) to your web app. Also supports Android in one codebase.

npm install @capacitor/core @capacitor/cli
npx cap init "Volume" "com.activecatalyst.volume" --web-dir ./www
# Copy index.html → www/index.html
npm install @capacitor/ios
npx cap add ios && npx cap copy ios
npx cap open ios
💡Capacitor's @capacitor/health-kit lets you write workout data to Apple Health.
4
Apache Cordova — Legacy Option
▶
npm install -g cordova
cordova create volume-app com.activecatalyst.volume Volume
cd volume-app
cp /path/to/index.html www/index.html
cordova platform add ios
cordova build ios
open platforms/ios/Volume.xcworkspace
📝Requires ios-deploy npm package and Xcode CLI tools (xcode-select --install).

Android is significantly more open than iOS — you can sideload APKs directly without any developer account or store. Four options from zero-friction to full Play Store.

1
PWA via Chrome — Install to Home Screen (Easiest)
▶

Android Chrome has first-class PWA support — far better than iOS Safari. The install prompt appears automatically once the app meets PWA criteria, no "Add to Home Screen" hunting required.

Step 1: Add the same manifest.json and service worker as the iOS PWA method (same files work on both platforms).

Step 2: Host on any HTTPS server. Open in Chrome on Android — Chrome will show an "Add Volume to Home Screen" banner automatically, or it appears under the Chrome menu.

Step 3: Once installed, the app launches full-screen with no browser UI, appears in the app drawer, and works offline via the service worker.

🟢Android PWAs support push notifications, background sync, and the Web Share API — features that iOS PWAs still lag on. This is the best quick-ship option for Android.
2
Capacitor — Build an APK (No Mac Required)
▶

Capacitor builds a real Android APK you can sideload or publish to the Play Store. Unlike iOS, you don't need a Mac — this works on Windows or Linux.

Requirements: Node.js, Android Studio, Java 17+, Android SDK.

# Install Capacitor
npm install @capacitor/core @capacitor/cli
npx cap init "Volume" "com.activecatalyst.volume" --web-dir ./www

# Copy your HTML file
mkdir www
cp /path/to/index.html www/index.html

# Add Android platform
npm install @capacitor/android
npx cap add android
npx cap copy android

# Open in Android Studio to build APK
npx cap open android

In Android Studio: Build → Build Bundle(s) / APK(s) → Build APK(s). The APK file path is shown in the notification — copy it to your phone and install directly.

💡To install the APK on your phone, enable Settings → Install Unknown Apps for your file manager. Android calls this "sideloading."

Add native Android plugins:

# Haptic feedback
npm install @capacitor/haptics

# Local notifications (workout reminders)
npm install @capacitor/local-notifications

# Google Fit integration
npm install @capacitor-community/google-fit

# In your JS:
import { Haptics, ImpactStyle } from '@capacitor/haptics';
await Haptics.impact({ style: ImpactStyle.Medium });
3
WebView App via Android Studio (Manual Wrapper)
▶

The Android equivalent of the WKWebView approach — wrap the HTML in a native Android shell app. More control than Capacitor, more setup required.

Step 1: In Android Studio: New Project → Empty Views Activity → Language: Kotlin.

Step 2: Copy index.html into app/src/main/assets/

Step 3: Replace MainActivity.kt:

import android.os.Bundle
import android.webkit.*
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
  override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    val webView = WebView(this)
    setContentView(webView)
    webView.settings.apply {
      javaScriptEnabled = true
      domStorageEnabled = true  // enables localStorage
      allowFileAccess = true
    }
    webView.loadUrl("file:///android_asset/index.html")
  }
}

Step 4: Add to AndroidManifest.xml inside the <application> tag:

<uses-permission
  android:name="android.permission.INTERNET"/>

<!-- Remove the action bar -->
<activity
  android:theme="@style/Theme.AppCompat.NoActionBar"
  ... />
⚠️domStorageEnabled = true is critical — without it localStorage is disabled and all workout data will be lost on close.
4
Play Store Distribution
▶

Publishing to the Google Play Store requires a one-time $25 developer registration fee (vs Apple's $99/yr). The process uses Android App Bundles (.aab) rather than APKs.

From Capacitor or Android Studio:

# Generate a signed release build via Capacitor
npx cap build android

# Or in Android Studio:
# Build → Generate Signed Bundle/APK
# → Android App Bundle → create keystore → build

Key requirements for Play Store submission:

# In build.gradle (app level):
android {
  defaultConfig {
    applicationId "com.activecatalyst.volume"
    minSdk 24          // Android 7.0+
    targetSdk 34       // latest stable
    versionCode 1
    versionName "1.0"
  }
}
🟢Play Store review is typically 1–3 days vs Apple's 1–7 days, and rejection rates are lower for simple utility apps. Store your keystore file somewhere safe — you can never change it once published.
